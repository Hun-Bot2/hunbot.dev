# Performance Risk Register

Reviewed: 2026-05-12

| Risk ID | Severity | Affected files | Build-time risk | Runtime/client-side risk | API/serverless risk | Recommended fix | Test or verification method | Status |
| --- | --- | --- | --- | --- | --- | --- | --- | --- |
| PERF-001 | High | `src/pages/[lang]/*.astro`, `src/pages/[lang]/blog/**/*.astro`, `src/utils/blog.ts` | Multiple pages call `getCollection('blog')`, filter, and sort independently. Build time grows with content volume and logic can drift. | None directly. | None. | Add shared helpers for language filtering, sorting, slug generation, categories, and series. | Unit tests for helpers; compare generated paths before and after refactor; run `npm run build`. | Planned |
| PERF-002 | High | `src/pages/[lang]/blog/index.astro`, `src/pages/[lang]/index.astro` | Listing pages render all posts for a language. Build output and page HTML grow as content grows. | Larger HTML and slower initial render for long archives. | None. | Add pagination for localized blog index and keep home page to a bounded recent-post set. | Build route count check; manual navigation through first, middle, and last pages. | Planned |
| PERF-003 | Medium | `src/pages/rss.xml.js`, `src/pages/sitemap.xml.ts` | Global RSS and custom sitemap iterate all posts. Link correctness is the larger issue, but content volume also grows output size. | Feed readers and crawlers receive larger documents. | If not fully prerendered, route generation can become dynamic work. | Fix links and decide whether to keep one global feed or add per-language feeds. Keep sitemap generation deterministic. | Validate feed XML and sitemap XML; crawl sample URLs; run build. | Planned |
| PERF-004 | Medium | `public/images/**`, MDX content, `BlogPost.astro`, listing pages | Public images bypass Astro image optimization unless manually prepared. Large images increase deploy size and transfer size. | Hero and inline images can be large on article pages and listing cards. | None. | Add content image guidelines: public paths allowed, but require sensible dimensions/compression and width/height where practical. Consider Astro image only as a targeted improvement. | Image inventory by size; Lighthouse/page weight checks on image-heavy posts. | Planned |
| PERF-005 | Medium | `src/components/BaseHead.astro` | None. | Google Fonts and KaTeX CSS are loaded from external resources on all pages, including pages without math. | None. | Load math CSS only where needed if feasible; review font strategy before adding more font packages or external font URLs. | Browser network audit on home, post with math, and post without math. | Planned |
| PERF-006 | Medium | `src/components/BaseHead.astro`, `Header.astro`, `FloatingLanguagePicker.astro`, `TableOfContents.astro`, `ViewCounter.astro` | None. | Many pages ship inline scripts for theme, menus, language picker, TOC, analytics, and view counts. | View counter script triggers API calls on post pages. | Move reusable behavior into smaller external modules only when it helps CSP and runtime cost; avoid adding new global scripts. | Lighthouse JS execution check; manual smoke test interactive controls. | Planned |
| PERF-007 | Medium | `src/pages/api/views.ts`, `ViewCounter.astro` | None. | Post pages perform a client-side POST to increment views. | Every post view can hit the serverless function and Redis unless duplicate suppression catches it. | Add stricter rate limiting, validate slugs, and consider GET-only display mode for crawlers or bots. | Serverless log sampling; Redis operation count; API tests for duplicate suppression. | Planned |
| PERF-008 | Low | `src/components/TableOfContents.astro` | None. | TOC script scans headings, manages scroll/drag state, and observes page behavior on every post. | None. | Keep TOC post-only. Avoid loading on non-article pages. Verify it remains hidden for posts without headings. | Manual post check with no headings, many headings, mobile, and desktop. | Planned |
| PERF-009 | Medium | `src/pages/[lang]/blog/categories.astro`, `src/pages/[lang]/blog/tags.astro` | Category/tag indexes scan all language posts at build time. This is fine now but grows with content and Library entries. | Category pages can grow dense without pagination or grouping. | None. | Reuse shared collection indexes. Add pagination or grouping only when content volume warrants it. | Build time baseline; generated category count diff. | Planned |
| PERF-010 | Medium | `public/scripts/neuralNetwork.js`, `public/scripts/controlsManager.js`, `public/scripts/themeManager.js` | None if dormant. | If loaded, the 3D visualization uses WebGL, fonts, particles, animation, and a global `THREE` dependency. | None. | Keep 3D scripts opt-in and route-scoped. Verify actual loading path before investing in optimization. | Network and performance profile on any page that loads the visualization. | Needs manual review |
| PERF-011 | High | Future Library collection/data files | Large Library datasets in Content Collections can slow builds and inflate serverless bundles. | Large client payloads can hurt search, filtering, and browsing. | If Library APIs are added, filtering can become runtime cost. | Separate approved public metadata from private candidates. Keep raw datasets out of the repo unless licensed and intentionally summarized. | Build with synthetic Library volume; bundle/page size check; schema validation. | Planned |
| PERF-012 | Medium | Future search implementation | Search indexing can add build time and deploy size. | Client-side search can add JS and index download cost. | None if static search. | Prefer static indexing such as Pagefind only after route and content boundaries are stable. Scope indexes by language/section. | Build-time index size check; query latency and page weight check. | Planned |
