Do we really need a translator when Google Translate already exists? That was my first thought. But I felt it would be better to organize blog posts by language, so I added English and Japanese pages. Since manual translation every time is both difficult and time-consuming, I wanted to run a strong model locally and implement translation without ongoing API costs. The thumbnail image shows the overall architecture diagram.

The workflow is as follows:
1. I write the blog post in Korean in VS Code (the file is stored locally, and I manually commit and push when I want).
2. I run the local LLM from the terminal and enter a translation command (monitoring and logging are mandatory).
3. The translated post is saved in my VS Code project under the /en and /jp folders.
4. While the AI reviews the output, I also review it to check translation quality.
5. I manually commit and push.

## Choosing a Local LLM
For English translation, I chose Ollama's gamma2 model, and for Japanese translation, I chose Qwen2.5:7b. I selected models that were rated highly based on community feedback and benchmark results.

I ran them overnight and checked the results in the morning, but the models caused problems. For example, in the Japanese and English versions of "Blog Devlog 05," the LLM added content it wanted to write on its own.

Because of these issues, I plan to discard all translated outputs first and then revise the logic.
Before anything else, I need to understand how each model behaves. Then I can decide whether to improve prompt engineering or split the Python code into more detailed steps.

I also think loading images separately for every translated post may be inefficient. I should complete the translation system first, then work on optimization.

## Grafana (LLM Monitoring)
![Grafana LLM Monitoring Dashboard](/images/BLOG/grafana01.png)

When running an LLM, monitoring is essential: how fast it runs, token throughput, and GPU usage. I used Grafana for that.
Grafana is an open-source monitoring tool that connects to many data sources and provides strong visualization. It was my first time using it, but it was easier than expected, so I got used to it quickly.
